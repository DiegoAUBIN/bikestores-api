🔐 Security & Authentication

Session-based Auth: Stored in browser sessionStorage or localStorage

API Key: Required for POST/PUT/DELETE operations, sent via X-API-KEY header

CSRF Protection: Session-based authentication mitigates CSRF attacks

Input Validation: Client-side validation in validateCrudOperation(), server-side validation on API

📦 Event Flow

  1. User logs in with email/password
  2. Session token stored locally, UI updated based on role
  3. User selects CRUD resource and operation
  4. Form fields appear/hide dynamically via syncCrudFieldsVisibility()
  5. User fills form and clicks submit
  6. handleCrudSubmit() validates operation and builds payload
  7. apiRequest() sends authenticated request with X-API-KEY header
  8. Response processed and UI updated accordingly
